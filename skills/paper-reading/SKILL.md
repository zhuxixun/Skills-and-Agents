# Paper Reading

## 何时激活

- 用户分享论文 PDF 或链接
- 用户要求总结论文
- 用户问"这篇论文讲了什么"
- 用户问"这个模型是怎么工作的"

## 工作流程

### Step 1: 获取论文内容

**优先顺序**：
1. **本地 PDF 文件**：先检查用户是否提供了本地 PDF 路径，如有则读取 PDF 内容
2. **WebFetch**：使用 WebFetch 获取在线论文内容
3. **用户粘贴**：让用户直接粘贴论文文本内容

### Step 2: 提取核心信息

按顺序提取以下内容：

#### 2.1 论文基本信息
- 标题、作者、机构、年份
- 发表 venue
- 论文试图解决的核心问题

#### 2.2 任务类型
- 分类、回归、生成、检索、排序、聚类、其他
- 任务的数学形式化定义
- 优化目标/损失函数

#### 2.3 模型架构（详尽）
- **输入层**：
  - 特征种类（sparse/dense、类别型/数值型）
  - 特征维度
  - 特征表征方式（embedding、归一化、hash bucket 等）
  - 是否共享嵌入、嵌入维度是否统一

- **主干网络结构**：
  - 完整的前向传播流程
  - 每层的输入输出维度
  - 关键计算步骤和公式

- **核心模块**（如适用）：
  - 各子模块的名称和功能
  - 模块间的连接方式
  - 重要模块的具体实现细节

#### 2.4 核心机制
- 关键算法的数学表达
- 核心创新点的具体实现方式
- 与之前方法的技术细节差异

#### 2.5 训练细节
- 优化器、学习率、batch size
- 正则化方式
- 收敛条件或训练轮数

#### 2.6 参数规模
- **总参数量级**：如 ~10M、~100M、~1B 等
- **各部分参数量分布**：embedding 层、主干网络、输出层等
- **计算量/推理延迟**（若论文给出）

#### 2.7 特殊依赖
- **硬件依赖**：CUDA/cuDNN/TensorRT/Transformer Engine 等 NVIDIA 加速库
- **框架依赖**：PyTorch/TensorFlow/JAX 等深度学习框架的特殊模块
- **第三方库**：如 NVIDIA Apex、DeepSpeed、Megatron-LM 等
- **若无特殊依赖**：注明"仅需标准深度学习框架（如 PyTorch/TensorFlow）"

#### 2.8 核心贡献
- 论文宣称的主要创新点
- 与之前方法的关键区别（技术细节层面）

### Step 3: 输出完整压缩文档

将上述所有信息整合为一份**完整的论文压缩文档**，格式如下：

```
# [论文标题]

## 基本信息
- 作者：
- 机构：
- 年份：
- 发表：

## 要解决的问题
[问题描述和背景]

## 任务定义
[数学形式化描述，包括输入、输出、优化目标]

## 模型架构

### 输入层
- 特征类型：
- 特征维度：
- 表征方式：
- 嵌入维度（若适用）：

### 主干网络
[完整的前向传播流程，包括每一步的输入输出]

### 核心模块（详述）
[关键模块的具体实现，包括公式]

## 核心机制
[核心算法/机制的技术细节描述]

## 训练细节
- 优化器：
- 学习率：
- 正则化：
- 其他：

## 参数规模
- 总参数量级：
- 各部分参数分布：
- 计算量/推理延迟（若给出）：

## 特殊依赖
- 硬件依赖（CUDA/cuDNN/TensorRT 等）：
- 框架依赖：
- 第三方库：
- 或标注"仅需标准深度学习框架"

## 核心贡献
1. [创新点1及具体技术细节]
2. [创新点2及具体技术细节]
3. ...

## 与之前方法的区别
[技术细节层面的对比，包括优缺点]
```

## 原则

1. **不用比喻** - 直接描述机制，不说"像...一样"
2. **精确描述** - 使用专业术语，给出具体的公式和维度
3. **信息完整** - 涵盖模型的技术细节，无需追问即可理解全貌
4. **对比基线** - 必要时说明与之前方法的技术差异
5. **忽略实验细节** - 不写实验配置、数据集统计、benchmark 数字等
6. **优先压缩技术细节** - 保留模型架构、算法、公式，压缩实验和调参内容

## 输出示例

### Good（完整压缩，信息无损失）

```
# DeepFM: A Factorization-Machine based Neural Network for CTR Prediction

## 基本信息
- 作者：Huifeng Guo, Ruiming Tang, et al.
- 机构：Harbin Institute of Technology, Huawei Noah's Ark Lab
- 年份：2017
- 发表：SIGIR

## 要解决的问题
点击率（CTR）预估需要同时建模低阶特征交互（如特征两两组合）和高阶特征交互，传统方法如 FM 仅能建模低阶，Wide&Deep 需要人工特征工程。

## 任务定义
- 输入：用户特征、广告特征、上下文特征（sparse + dense）
- 输出：点击概率 (0-1)
- 优化目标：交叉熵损失 / log loss

## 模型架构

### 输入层
- 特征类型：sparse features（类别型）+ dense features（数值型）
- sparse 特征经 embedding 转为 k 维稠密向量
- dense 特征经归一化处理
- 所有特征 embedding 拼接为原始输入向量 x₀

### 主干网络
1. **Embedding 层**：对每个 sparse 特征 x_i → v_i ∈ ℝᵏ
2. **FM 层**：建模二阶特征交互，输出 ΣᵢΣⱼ<vᵢ, vⱼ>xᵢxⱼ
3. **Deep 层**：DNN(concat(embeddings)) 建模高阶交互
4. **输出层**：Sigmoid(FM_output + DNN_output)

## 核心机制
- FM 层：显式建模二阶特征交互，公式为 y_FM = ΣᵢΣⱼ<xᵢ, xⱼ>vᵢ⊙vⱼ
- DNN 层：通过多层非线性变换隐式建模高阶交互
- 联合训练：FM 和 DNN 共享相同的特征嵌入

## 训练细节
- 优化器：Adam
- 学习率：0.001
- 正则化：dropout + L2

## 核心贡献
1. 同时建模低阶（FM）和高阶（DNN）特征交互，无需人工特征工程
2. FM 和 DNN 共享特征嵌入，端到端训练
3. 兼具 FM 的可解释性和 DNN 的强表达能力

## 与之前方法的区别
| 方法 | 低阶交互 | 高阶交互 | 特征工程 |
|------|---------|---------|---------|
| FM | ✓ | ✗ | 需要 |
| Wide&Deep | ✓（Wide） | ✓ | 需要 |
| DeepFM | ✓（FM） | ✓（DNN） | 不需要 |
```

### Bad（信息损失过多，仍需追问）

```
DeepFM 是一个点击率预估模型，用 FM 建模低阶特征交互，用 DNN 建模高阶交互。
（错误：没有说明输入特征类型、嵌入维度、特征如何处理、FM 和 DNN 如何连接）
```
